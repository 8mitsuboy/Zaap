# Zaap プロジェクト ルール

## 画像の配置ルール

### 記事内画像・サムネイル
- 画像はすべて `public/posts/{記事ディレクトリ名}/` に配置する
- `src/pages/posts/` 内には画像を置かない

### markdown内の参照パス
- 絶対パスで記述する: `/posts/{記事ディレクトリ名}/{ファイル名}`
- 相対パス（`./image.png`）は使わない

### frontmatter のサムネイル設定
```yaml
image:
  url: /posts/{記事ディレクトリ名}/{ファイル名}
  alt: "画像の説明"
```

## 記事の構成
- ブログ記事: `src/pages/posts/{日付-スラッグ}/index.md`
- レイアウト: `../../../layouts/MarkdownPostLayout.astro`

## 学習・アウトプット方針

### 目標
- **なりたい状態**: 3Dオブジェクトを作ってWebやアプリに組み込める
  - フロントエンドエンジニアとして3Dオブジェクトを使ったサイト作成や、3Dオブジェクト作成の案件を受注できる
- **直近のマイルストーン**: AstroサイトにBlenderで作った3Dを
  組み込んだポートフォリオを完成させる
- **さらに先**: 自作モバイルアプリに3Dキャラクターを登場させる

### ブログで扱う技術領域
Blender / Flutter / React / DDD など、上記目標に向けた学びを幅広く記録する

### ブログ運用方針（zaap-dev.netlify.app）
- **記事の粒度**: 1要求1記事。「〇〇したい」「〇〇でつまった」単位で切る
- **テンプレートの使い分け**:
  - inputを急いでいるとき → `cornell`（ログモード）。タイトルも雑でいい、画像不要
  - 整理して伝えたいとき → `article`（読まれるモード）
- **スラッシュコマンド**: 記事作成時はタイトルだけ聞いて、スラッグとタグはAIが提案する

### 復習
- 忘れた・詰まったときに過去記事を読み返す形で対応
- 定期的な振り返りの仕組みは今のところ不要
